# SKAT-ESP32-Tenzo

Прошивка контроллера стенда **SKAT-Tenzo**: измерение силы тяги (HX711), тока и напряжения шины (INA226), управление приводами по стандартному RC-PWM и передача телеметрии по **Bluetooth LE**.

Сопутствующее Android-приложение для мониторинга и управления: [SKAT-Load-cell](https://github.com/reduct0r/SKAT-Load-cell)

## Совместимое оборудование

### Микроконтроллер (целевая платформа)

| Параметр | Значение |
|----------|----------|
| **Платформа сборки** | `espressif32`, board `esp32dev` |
| **Микроконтроллер** | **ESP32** (серия ESP32-D0WD, ESP32-WROOM-32 и аналоги) |
| **Ядра / частота** | Dual-core, 240 MHz |
| **Bluetooth** | BLE 4.2 (обязательно) |
| **Flash** | ≥ 4 MB (рекомендуется; используется схема разделов `huge_app`) |
| **Framework** | Arduino (ESP-IDF через PlatformIO) |

**Поддерживается штатно:** ESP32 DevKitC, NodeMCU-32S, DOIT ESP32 DevKit V1 и другие платы класса **ESP32-WROOM-32** с выведенными GPIO 18, 19, 21, 22, 25, 26.

**Не поддерживается без доработки:**

| Чип / плата | Причина |
|------------|---------|
| **ESP8266** | отсутствует BLE |
| **ESP32-S2** | отсутствует BLE |
| **ESP32-C3 / ESP32-H2** | иной набор GPIO; требуется смена `board` и проверка распиновки |
| **ESP32-S3** | возможна адаптация, но конфигурация по умолчанию не тестировалась |

### Датчики и модули

| Устройство | Интерфейс | Примечание |
|------------|-----------|------------|
| **HX711** + тензoдатчик (load cell) | 2-wire serial (DT/SCK) | Любой модуль HX711 3.3 V |
| **INA226** | I2C | Адрес 0x40–0x4F; автоопределение. Поддерживаются совместимые клоны с корректным Manufacturer ID |
| **ESC / RC-регулятор** | PWM 50 Hz | Стандартный **servo PPM** (см. раздел [Приводы](#приводы-esc)) |

## Среда разработки

### Рекомендуемый вариант: VS Code + PlatformIO

1. Установите [Visual Studio Code](https://code.visualstudio.com/).
2. Установите расширение **[PlatformIO IDE](https://marketplace.visualstudio.com/items?itemName=platformio.platformio-ide)** (`platformio.platformio-ide`).
3. Клонируйте репозиторий и откройте папку проекта в VS Code.
4. PlatformIO автоматически загрузит toolchain **xtensa-esp32**, framework **arduino-espressif32** и библиотеки из `platformio.ini`.

### Альтернатива: PlatformIO Core (CLI)

```bash
pip install platformio
cd SKAT-ESP32-Tenzo
pio run
```

### Версии (ориентир из `platformio.ini`)

| Компонент | Версия |
|-----------|--------|
| Platform | `espressif32` (PlatformIO) |
| Board | `esp32dev` |
| Framework | `arduino` |
| HX711 | bogde/HX711 ^0.7.5 |
| INA226 | robtillaart/INA226 ^0.6.6 |

Точные версии toolchain фиксируются PlatformIO при первой сборке.

## Сборка и прошивка

```bash
# Сборка
pio run

# Прошивка (указать свой COM-порт)
pio run -t upload --upload-port COM3

# Монитор Serial (115200 baud)
pio device monitor --port COM3 --baud 115200
```

**Локальный COM-порт:** создайте файл `platformio_local.ini` (не коммитится):

```ini
[env:esp32dev]
upload_port = COM3
monitor_port = COM3
```

После включения питания прошивка выводит минимальный импульс **2,5 с** для инициализации ESC, затем начинает BLE-рекламу с именем **`SKAT-Tenzo`**.

## Распиновка по умолчанию

Соответствие маркировке **DevKit (Dxx = GPIO xx)**:

| Функция | GPIO | DevKit | Направление | Описание |
|---------|------|--------|-------------|----------|
| HX711 DT (DOUT) | **18** | D18 | вход | Данные тензодатчика |
| HX711 SCK | **19** | D19 | выход | Тактовый HX711 |
| I2C SDA | **21** | D21 | I/O | Шина I2C (INA226) |
| I2C SCL | **22** | D22 | выход | Шина I2C (INA226) |
| ESC канал A | **25** | D25 | выход | RC-PWM привод 1 |
| ESC канал B | **26** | D26 | выход | RC-PWM привод 2 |
| — | 3.3 V | 3V3 | питание | HX711, INA226 (логика) |
| — | GND | GND | — | **Общая земля** ESP32, ESC, датчиков, БП |

> **Важно:** GND ESP32, сигнальных (чёрных) проводов ESC и минуса силового питания должны быть объединены.

## Подключение датчиков

### Тензодатчик (HX711)

```
Load cell          HX711 module        ESP32
  E+  ────────────  E+                   
  E−  ────────────  E−                   
  A+  ────────────  A+                   
  A−  ────────────  A−                   
                    VCC  ───────────────  3.3 V
                    GND  ───────────────  GND
                    DT   ───────────────  GPIO 18
                    SCK  ───────────────  GPIO 19
```

### Датчик тока и напряжения (INA226)

Типовая схема — **high-side** (шунт в разрыве «+» силовой линии):

```
BAT+ ──── VIN+ (INA226)
              │
         IN+ ─┤ 
         шунт R
         IN− ─┤
              │
         нагрузка / ESC (+)

BAT− / GND ── GND (INA226) ── GND (ESP32)

SDA ── GPIO 21
SCL ── GPIO 22
```

| Вывод INA226 | Подключение |
|--------------|-------------|
| **VIN+** | Плюс батареи / БП (до шунта) — измерение **U шины** |
| **IN+** | Со стороны источника относительно шунта |
| **IN−** | Со стороны нагрузки относительно шунта |
| **GND** | Общая земля с ESP32 |
| **SDA / SCL** | I2C, GPIO 21 / 22 |

- Ток: **I = ΔUшунта / Rшунта** (значение R задаётся через BLE / NVS).
- Значение шунта по умолчанию в прошивке: **0,005 Ω** (5 mΩ); фактическое сопротивление указывается при калибровке в приложении.

## Приводы (ESC)

Прошивка формирует сигнал управления по **стандарту RC-сервопривода (PPM/PWM)**. Поддерживается большинство ESC для моделей, рассчитанных на приём PWM с приёмника, в том числе:

- SimonK, BLHeli, BLHeli_32, AM32 и аналоги;
- регуляторы авиа/авто линейки с входом «Throttle / Signal».

### Параметры сигнала (по умолчанию)

| Параметр | Значение |
|----------|----------|
| Частота кадра | **50 Hz** (период 20 ms) |
| Минимальный импульс | **1000 µs** — останов / idle |
| Максимальный импульс | **2000 µs** — полная мощность |
| Разрешение LEDC | 14 bit |
| Каналы | **2** (GPIO 25 и GPIO 26), **одинаковый** сигнал на оба |

Диапазон импульса может быть изменён в коде / NVS (`esc_min`, `esc_max` в namespace `skat`).

### Не поддерживается

- **OneShot125 / OneShot42 / Multishot** — цифровые протоколы без PWM;
- **DShot** — битовый протокол без отдельной линии PWM;
- ESC с иным стандартом (например, 400 Hz PWM) без изменения `ESC_FREQ_HZ` в `motor_module.cpp`.

### Логика безопасности

- При **disarm** или **разрыве BLE** — немедленный переход на минимальный импульс.
- Команда **set_pwm** принимается только в состоянии **arm**.
- При старте — удержание минимального импульса **2,5 с** (инициализация ESC).

### Подключение ESC

```
ESP32 GPIO 25 ── белый/жёлтый (signal) ESC A
ESP32 GPIO 26 ── белый/жёлтый (signal) ESC B
ESP32 GND     ── чёрный (GND) ESC
```

Силовые провода ESC (BAT+/BAT−) — к источнику питания стенда

## Адаптация под другое оборудование

Все назначения GPIO задаются **константами в исходном коде**).

| Задача | Файл | Константы |
|--------|------|-----------|
| Пины HX711 | `src/main.cpp` | `PIN_HX711_DT`, `PIN_HX711_SCK` |
| Пины I2C | `src/main.cpp` | `PIN_I2C_SDA`, `PIN_I2C_SCL` |
| Пины ESC, частота PWM | `src/motor_module.cpp` | `PIN_MOTOR_A/B`, `ESC_FREQ_HZ`, `ESC_DEFAULT_MIN_US`, `ESC_DEFAULT_MAX_US` |
| Число каналов ESC | `src/motor_module.cpp` | второй канал можно отключить, закомментировав `ledcAttachPin` / `ledcWrite` для GPIO 26 |
| Имя BLE | `src/main.cpp` | `BLE_DEVICE_NAME` |
| UUID GATT | `src/ble_module.cpp` | `kServiceUuid`, `kTelemetryUuid`, … |
| Другая плата ESP32 | `platformio.ini` | `board = …` |

При смене платы (ESP32-S3 и т.д.) проверьте: наличие BLE, доступность выбранных GPIO, совместимость с Arduino-LEDC для PWM.

## Bluetooth LE

### Общие параметры

| Параметр | Значение |
|----------|----------|
| Имя устройства | `SKAT-Tenzo` |
| Service UUID | `a1b2c3d4-e5f6-7890-abcd-ef1234567890` |
| Telemetry UUID | `a1b2c3d4-e5f6-7890-abcd-ef1234567891` — READ, NOTIFY |
| Command UUID | `a1b2c3d4-e5f6-7890-abcd-ef1234567892` — WRITE, WRITE_NR |
| Response UUID | `a1b2c3d4-e5f6-7890-abcd-ef1234567893` — READ, NOTIFY |
| Период телеметрии | **500 ms** |

### Телеметрия (JSON, NOTIFY)

Пример полей: `force_g`, `force_n`, `current_a`, `bus_v`, `shunt_mv`, `shunt_ohm`, `shunt_ext`, `hx711_ok`, `ina226_ok`, `scale`, `motors_armed`, `motor_pwm_pct`, `esc_pulse_us`, `bus_v_scale_e4`, …

Запрос снимка: команда `get` или `data`.

### Команды (JSON → Command characteristic)

| Команда | Параметры | Назначение |
|---------|-----------|------------|
| `get` / `data` | — | Получить текущую телеметрию |
| `tare` | — | Обнулить HX711, сохранить offset |
| `calibrate` | `grams` | Калибровка по эталонной массе |
| `set_scale` | `scale` | Задать коэффициент вручную |
| `reset` | — | Сброс scale и обнуление |
| `set_force_sign` | `sign` (1 / −1) | Знак силы |
| `set_shunt` | `ext_ohm`, … | Параметры шунта |
| `recal_ina226` | — | Перекалибровка INA226 |
| `calibrate_bus_v` | `ref_v` | Калибровка напряжения шины |
| `zero_current` | — | Обнуление тока |
| `set_current_sign` | `sign` (1 / −1) | Знак тока |
| `arm` | — | Разрешить PWM |
| `disarm` | — | Запрет PWM, мин. импульс |
| `set_pwm` | `pct` (0…100) | Мощность привода (только armed) |

Ответ на команды — JSON в **Response** (notify). Тяжёлые команды (`tare`, `calibrate`, …) выполняются в `loop()`; `arm`, `disarm`, `set_pwm` — без задержки в callback.

## Калибровка и NVS

Параметры хранятся в **NVS** (namespace `skat`):

| Ключ | Описание |
|------|----------|
| `scale` | Коэффициент HX711 |
| `offset` | Нулевая точка HX711 |
| `force_sign` | Знак силы |
| `shunt_ext`, `shunt_brd`, `shunt_brd_inc` | Шунт INA226 |
| `i_zero_a`, `i_sign` | Обнуление и знак тока |
| `bus_v_scale` | Масштаб напряжения шины |
| `ina_addr` | Адрес INA226 на I2C |
| `esc_min`, `esc_max` | Диапазон импульса ESC |

Калибровка выполняется через [Android-приложение](https://github.com/reduct0r/SKAT-Load-cell) или прямые BLE-команды.
